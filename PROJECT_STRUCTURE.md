## Team-Specific File Locations

### ?? Frontend/UI Team
**Primary Directory**: `src/frontend/ui/`

**Key Files**:
- `main_window.h/.cpp` - Application shell (start here)
- `code_editor.h/.cpp` - C++ code editor
- `visualization_pane.h/.cpp` - Main visualization canvas
- `control_panel.h/.cpp` - Playback controls
- `diagnostic_output.h/.cpp` - Logs and metrics
- `memory_viewer.h/.cpp` - Low-level memory view

**Build File**: `src/frontend/CMakeLists.txt`

**Implementation Guide**: `docs/UI_GUIDE.md`

---

### ?? C2V Team (Code-to-Visualization)
**Primary Directory**: `src/analysis/`

**Subdirectories**:
- `clang_integration/` - AST parsing and structure extraction
- `instrumentation/` - LLVM pass for runtime instrumentation
- `state_mapping/` - Execution logging and state conversion

**Key Files**:
- `clang_integration/ast_parser.h/.cpp` - Parse C++ with Clang (start here)
- `clang_integration/structure_extractor.h/.cpp` - Identify data structures
- `instrumentation/llvm_instrumentation.h/.cpp` - Inject logging callbacks
- `state_mapping/execution_logger.h/.cpp` - Capture execution events
- `state_mapping/state_mapper.h/.cpp` - Convert events to visualization states

**Build File**: `src/analysis/CMakeLists.txt`

**Implementation Guide**: `docs/C2V_GUIDE.md`

---

### ?? V2C Team (Visualization-to-Code)
**Primary Directory**: `src/synthesis/`

**Subdirectories**:
- `code_generation/` - Code generation and rewriting
- `api_mapping/` - User API tracking and mapping

**Key Files**:
- `code_generation/clang_rewriter.h/.cpp` - Source code rewriting (start here)
- `code_generation/ast_generator.h/.cpp` - Generate C++ code
- `code_generation/code_formatter.h/.cpp` - Code formatting (clang-format)

## Build System Organization

### CMakeLists.txt Hierarchy

```
CMakeLists.txt (root)
??? Creates dataviz-app executable
??? Links to all libraries:
?   ??? frontend_engine (Qt GUI)
?   ??? analysis_engine (C2V)
?   ??? synthesis_engine (V2C)
?   ??? core_lib (shared)
?   ??? visualization_engine
?
??? Calls add_subdirectory() for each component:
    ??? src/frontend/CMakeLists.txt
 ??? src/analysis/CMakeLists.txt
    ??? src/synthesis/CMakeLists.txt
    ??? src/core/CMakeLists.txt
    ??? src/visualization/CMakeLists.txt
```

Each component's CMakeLists.txt:
1. Defines its own library sources
2. Sets include directories
3. Links to dependencies
4. Exports to parent CMake

### Building Components Independently

Although all components are built together, the structure allows:

```bash
# Build entire project
cmake --build . --config Release

# Rebuild specific library (after changing files)
cmake --build . --config Release --target frontend_engine
cmake --build . --config Release --target analysis_engine

# Run specific component tests
ctest -R "frontend" --verbose
```

## File Naming Conventions

### Header Files (.h)
- Use `.h` extension for all C++ headers
- Full names: `visualization_pane.h` (not `vis_pane.h`)
- Match implementation file name: `foo.h` ? `foo.cpp`

### Implementation Files (.cpp)
- Use `.cpp` extension for all implementations
- Match header file name
- Include corresponding header first

### CMakeLists.txt
- Uppercase `CMakeLists.txt` (exact spelling)
- One per directory
- Define SOURCES variable listing all files

## Header Guard Convention

All headers use `#ifndef` style guards:
```cpp
#ifndef COMPONENT_NAME_H
#define COMPONENT_NAME_H

// Content

#endif // COMPONENT_NAME_H
```

Where `COMPONENT_NAME` matches the file stem with underscores:
- `code_editor.h` ? `CODE_EDITOR_H`
- `graphviz_layout_engine.h` ? `GRAPHVIZ_LAYOUT_ENGINE_H`

## Include Path Organization

### System Includes
```cpp
#include <vector>
#include <string>
#include <map>
```

### Qt Includes
```cpp
#include <QWidget>
#include <QMainWindow>
```

### LLVM/Clang Includes
```cpp
#include "clang/Frontend/CompilerInstance.h"
#include "llvm/IR/Module.h"
```

### Project Includes
Relative to component root (set by CMake):
```cpp
#include "ast_parser.h"  // Same directory
#include "../utilities/logger.h"  // Different component
```

## Adding New Files

### When adding a new component file:

1. **Create in correct directory** following team organization
2. **Update component CMakeLists.txt** to include new file
3. **Use proper header guards** with COMPONENT_NAME_H format
4. **Include in correct order**:
   - Corresponding header first
   - System includes
   - Qt includes
   - Project includes
5. **Follow naming conventions**: camelCase for implementation, descriptive names
6. **Commit to feature branch** with clear message

### Example: Adding new C2V analysis component

```bash
# Create file in correct directory
touch src/analysis/clang_integration/my_new_analyzer.h
touch src/analysis/clang_integration/my_new_analyzer.cpp

# Edit src/analysis/CMakeLists.txt:
# Add to ANALYSIS_SOURCES:
#   clang_integration/my_new_analyzer.h
#   clang_integration/my_new_analyzer.cpp

# Use in C++:
#include "my_new_analyzer.h"
```
